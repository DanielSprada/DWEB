Software de versionamento

Um software de versionamento é um programa que visa auxiliar os desenvolvedores a controlar as versões dos códigos e programas que estão desenvolvendo. Em trabalhos com equipes, auxilia também a todos estarem alinhados e com acesso ao código atualizado, visto que, o mesmo pode ser salvo junto com seu versionamento em nuvem.

No software de versionamento é possível fazer a sincrozinação entre repositórios locais e online. Pode-se também fazer a atualização de versões por usuários, Ex.: Armando e Bruno estão trabalhando no código, sendo assim podemos ter versões como: V.02.A; V.02.B, e também mantém uma versão corrente atualizada para todos os usuários.

O software também alerta e mostra quais foram as alterações feitas por cada usuário, mantendo um melhor controle do projeto.

Possuímos dois tipos de softwares de controle de versão:

  • Centralizado
  • Distribuído

Central: Possui repositório central em um servidor com controle das versões, é dependente do servidor para funcionar. Um exemplo desse tipo de software é o Subversion.

Distribuído: possui um repositório local onde, as alterações são realizadas e salvas localmente e posteriormente subidas ao servidor, no repositório central. Não depende do servidor. Exemplos de softwares desse tipo são o Mercurial e o Git.

Ferramentas que utilizaremos:

GIT
Software de livre versionamento, criado em 2005 por Linus Torvalds. Permite registrar o histório completo de versionamentos, alterações e revisões em um projeto, faz a sincronia em servidor o que permite maior segurança e trabalho em equipe, porém, não necessita de acesso a internet e ao servidor central.
GitHub
Plataforma de hospedagem de código fonte e arquivos com controle de versão, pode ser sincronizado ao GIT e salvar em nuvem todo um projeto, bem como, seu versionamento.
GitHub Desktop
Permite o controle de versões usando o software com interface gráfica, possui o mesmo papel do GIT, porém, por possuir interface gráfica acaba sendo um processo mais fácil de executar.
Visual Studio Code
Editor de código que temos utilizado na disciplina. Possui total integração com o GIT e GitHub.

Comandos GIT

commit
torna permanente um conjunto de alterações em um repositório.
push
envia alterações para o servidor no repositório remoto.
fetch
verifica se o seu repositório local está em sincronia com o remoto.
pull
sincroniza o repositório local com o remoto atualizando as versões.

Utilização do GitHub Desktop

Você pode criar um repositório e utilizar licenças privadas ou públicas.
Adicionar um repositório já existente em seu computador (caso ele não esteja no formato do GIT, terá a opção de "formatá-lo" no formato correto para o GIT).
Ou então, clonar um repositório da internet para sua máquina local.

É importante sempre lembrar ao editar um código, se deseja que o mesmo seja atualizado no repositório realizar o commit, inserindo uma breve descrição das alterações que realizou.
Após isso, para enviar ao repositório no servidor, precisa clicar em push.
Se deseja buscar por diferenças entre seu repositório local e o online clica em fetch e se deseja puxar o que está remoto e sincronizar com o local, realiza um pull.
